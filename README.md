# Projeto-To-Do-List-em-Python
Este projeto é um projeto feito totalmente em python feito para o terminal que é basicamente um gerenciador de tarefa podendo fazer de tudo adicionar tarefas edita-las e entre outras coisas/ foi feto no âmbito de um projeto escolar 


# enunciado da tarefa 
Mini-projeto prático de Python
Curso: Técnico/a Especialista em Tecnologias e Programação de Sistemas de Informação

1. Contextualização
No final da Unidade Curricular de Python, pretende-se que os formandos desenvolvam uma To-Do List em linha de comandos.

A aplicação deverá permitir ao utilizador criar, consultar, alterar, organizar e remover tarefas, aplicando os principais conteúdos trabalhados ao longo da UC.

Este projeto tem como objetivo consolidar conhecimentos de programação em Python através da criação de uma aplicação funcional, organizada e adequada ao tempo disponível.

2. Objetivo do Projeto
Deves desenvolver uma aplicação em Python, executada através da consola/terminal, que permita ao utilizador gerir uma lista de tarefas.

A aplicação deve disponibilizar menus claros, permitir a interação com o utilizador e apresentar mensagens compreensíveis em português.

No final do projeto, a aplicação deverá permitir gerir tarefas de forma simples, coerente e funcional.

3. Dados de uma Tarefa
Cada tarefa deve ter, pelo menos, os seguintes dados:

Identificador único;
Título;
Descrição;
Prioridade;
Estado;
Data de criação;
Data limite ou prazo;
A aplicação deve permitir trabalhar com diferentes estados de tarefa, por exemplo:

Pendente;
Em curso;
Concluída;
Cancelada.
4. Funcionalidades Obrigatórias
A aplicação deve incluir obrigatoriamente as seguintes funcionalidades:

Criar uma nova tarefa;
Listar todas as tarefas registadas;
Consultar os detalhes de uma tarefa específica;
Alterar dados de uma tarefa;
Marcar uma tarefa como concluída;
Remover uma tarefa;
Pesquisar tarefas por texto;
Filtrar tarefas por estado;
Apresentar estatísticas simples sobre as tarefas registadas.
As estatísticas devem incluir, pelo menos:

Número total de tarefas;
Número de tarefas pendentes;
Número de tarefas concluídas;
Número de tarefas por categoria;
Número de tarefas por prioridade.
5. Requisitos Técnicos
O projeto deve ser desenvolvido em Python e deve evidenciar a utilização dos principais conteúdos abordados ao longo da UC, nomeadamente:

Variáveis;
Operações com variáveis;
Estruturas condicionais;
if;
match;
Ciclos for e while;
Collections: list, dict, set e tuple;
Slicing de listas;
List comprehensions;
Funções;
Classes;
Organização do projeto em múltiplos ficheiros;
Utilização de *args;
Utilização de **kwargs.
A aplicação deve apresentar uma estrutura organizada e coerente, adequada a um projeto desenvolvido ao longo de uma semana.

Não é obrigatória a persistência de dados após o encerramento da aplicação, salvo indicação em contrário pelo formador.

6. Regras do Projeto
A aplicação deve ser executada através da consola/terminal.

O utilizador deve conseguir interagir com a aplicação através de menus claros e compreensíveis.

A aplicação deve validar os dados introduzidos pelo utilizador sempre que necessário.

A aplicação não deve terminar inesperadamente perante dados inválidos.

As mensagens apresentadas ao utilizador devem ser claras e escritas em português.

O código deve estar organizado, indentado e fácil de compreender.

A pasta do projeto deve conter apenas ficheiros relevantes para o funcionamento e avaliação do trabalho.

7. Funcionalidades Complementares
As seguintes funcionalidades são opcionais e servem para valorizar o projeto, desde que as funcionalidades obrigatórias estejam corretamente implementadas:

Ordenar tarefas por prioridade;
Ordenar tarefas por prazo;
Identificar tarefas em atraso;
Criar um histórico de alterações;
Confirmar ações críticas antes da sua execução;
Gerar um relatório final com o resumo das tarefas;
Criar uma experiência de utilização mais cuidada no terminal.
8. Entrega
O formando deve fazer a entrega através do GitHub Classroom e da tarefa criada no Moodle.

Documentos e ficheiros a entregar:

O ficheiro principal da aplicação;
Os restantes ficheiros Python necessários;
Eventuais ficheiros auxiliares utilizados pela aplicação;
Um pequeno ficheiro README.txt ou README.md com:
Nome do projeto;
Nome do formando;
Breve descrição da aplicação;
Instruções simples para executar o programa.
Na última sessão de formação será feita uma apresentação do trabalho desenvolvido.

9. Avaliação
A avaliação terá em conta:

Cumprimento dos requisitos do enunciado;
Funcionamento correto da aplicação;
Utilização adequada dos conteúdos abordados na UC;
Organização do código;
Clareza dos menus e mensagens;
Validação de dados;
Qualidade geral da solução apresentada;
Autonomia e capacidade de resolver problemas;
Criatividade nas funcionalidades adicionais;
Apresentação do trabalho na última sessão de formação.
10. Observações Finais
O projeto deve demonstrar que o formando é capaz de construir uma aplicação funcional em Python, aplicando de forma integrada os conceitos estudados ao longo da UC.

Mais do que criar uma aplicação extensa, pretende-se uma aplicação bem estruturada, coerente, funcional e adequada ao prazo definido para o projeto.
